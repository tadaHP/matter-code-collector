# Matter Code Collector 제품 요구사항

## 목적

Matter 기기의 QR 인식값과 숫자 코드를 개인용 웹앱에서 안전하게 등록, 조회, 수정, 삭제할 수 있게 한다. 사용자는 데스크톱과 모바일 브라우저에서 기기 정보를 빠르게 찾고, 필요한 순간 QR 코드와 숫자 코드를 다시 확인할 수 있어야 한다.

이 문서는 구현 상세 명세가 아니라 제품 요구사항 하네스다. 실제 API, 타입, DB 라이브러리, ORM, 테스트 프레임워크 선택은 다음 구현 계획에서 결정한다.

## 기술 및 운영 기준

- 프로젝트는 Next.js 16 App Router 기반으로 유지한다.
- 코드 작성 전 `node_modules/next/dist/docs/`의 관련 가이드를 확인하고, 현재 버전의 변경점과 deprecation notice를 따른다.
- UI는 Tailwind CSS v4를 사용해 데스크톱과 모바일을 모두 고려한 반응형 화면으로 만든다.
- 데이터 저장소는 SQLite를 사용한다.
- SQLite DB 파일 경로는 루트 `.env*` 파일의 서버 전용 환경변수 `MATTER_SQLITE_PATH`로 설정한다.
- `/src` 폴더를 사용하는 프로젝트이므로 `.env*` 파일은 `src/`가 아니라 프로젝트 루트에 둔다.
- 개발 환경과 개인 운영 환경에서 서로 다른 DB 경로를 설정할 수 있어야 한다.
- 인증 및 세션 보호에 필요한 서버 전용 비밀값은 `AUTH_SECRET` 환경변수로 관리한다.

## 사용자와 보안

- v1은 개인 운영을 전제로 하되, QR/Matter 코드 정보 보호를 위해 계정 로그인 기반 접근 제어를 제공한다.
- 공개 회원가입은 v1 범위 밖이다.
- 초기 관리자 계정 생성 방식은 구현 단계에서 결정한다.
- 사용자 비밀번호는 평문 저장을 금지하고, 안전한 해시로 저장해야 한다.
- 로그인 세션은 서버에서 검증 가능해야 하며, 모든 등록, 조회, 수정, 삭제 기능은 로그인한 사용자만 사용할 수 있어야 한다.
- 인증되지 않은 사용자는 로그인 화면 외의 기기 정보 화면과 데이터 API에 접근할 수 없어야 한다.

## 핵심 화면

- 로그인 화면
  - 계정 정보로 로그인한다.
  - 로그인 실패 시 과도한 내부 정보를 노출하지 않는 오류 메시지를 보여준다.
- 기기 목록 화면
  - 등록된 Matter 기기를 최근 등록순 또는 최근 수정순으로 볼 수 있다.
  - 별명, 실제 기기 이름, 제조사, 모델, 위치, 태그, 숫자 코드 일부를 스캔하기 쉬운 형태로 표시한다.
  - 데스크톱에서는 정보 밀도를 높이고, 모바일에서는 카드 또는 단일 열 중심으로 조작하기 쉽게 만든다.
- 기기 등록 화면
  - 카메라로 QR 코드를 스캔해 QR 인식값을 채울 수 있다.
  - 카메라를 사용할 수 없거나 권한이 거부된 경우 수동 입력으로 등록할 수 있다.
  - 숫자 코드, 별명, 실제 기기 이름, 기타 메타데이터를 입력할 수 있다.
- 기기 상세 화면
  - 저장된 QR 인식값을 QR 코드로 직접 표시한다.
  - QR 인식값과 숫자 코드를 복사할 수 있다.
  - 기기 정보 수정과 삭제 진입점을 제공한다.
- 기기 수정 화면
  - 기존 등록 정보를 수정할 수 있다.
  - QR 인식값과 숫자 코드 변경은 실수 방지를 위해 명확히 확인 가능한 UI로 제공한다.
- 삭제 확인
  - 삭제 전 대상 기기를 확인시키고, 삭제 후 목록으로 돌아간다.

## 핵심 기능

- Matter 기기 등록, 조회, 수정, 삭제를 제공한다.
- QR 인식값을 기반으로 QR 코드를 직접 표시한다.
- QR 인식값과 숫자 코드를 클립보드에 복사할 수 있다.
- 카메라 QR 스캔 등록을 지원한다.
- 카메라 QR 스캔은 모바일과 데스크톱 브라우저를 고려한다.
- 브라우저 카메라 기능은 HTTPS 또는 localhost 같은 secure context 요구사항이 있음을 구현 문서와 UI 흐름에서 고려한다.
- 카메라 사용이 불가능한 환경을 위해 수동 입력 fallback을 제공한다.
- 별명, 실제 기기 이름, 제조사, 모델, 위치, 태그, 메모로 검색 또는 필터링할 수 있다.
- 태그별, 위치별 필터를 제공한다.
- 최근 등록순 또는 최근 수정순 정렬을 제공한다.
- 동일한 QR 인식값 또는 숫자 코드가 이미 등록된 경우 중복 가능성을 알려준다.
- 등록 데이터의 백업 또는 내보내기 기능을 제공한다.

## 데이터 요구사항

### MatterDevice

향후 구현 시 Matter 기기 데이터는 제품 수준에서 다음 정보를 다룰 수 있어야 한다.

- `alias`: 사용자가 빠르게 식별하기 위한 기기 별명 또는 약칭.
- `deviceName`: 실제 기기 이름.
- `qrPayload`: Matter QR 코드에서 읽은 원본 인식값.
- `numericCode`: Matter 숫자 코드.
- `manufacturer`: 제조사.
- `model`: 모델명.
- `location`: 설치 위치 또는 보관 위치.
- `tags`: 검색과 필터링을 위한 태그 목록.
- `notes`: 기타 메모.
- `createdAt`: 등록 일시.
- `updatedAt`: 마지막 수정 일시.

### UserAccount

향후 구현 시 계정 데이터는 제품 수준에서 다음 정보를 다룰 수 있어야 한다.

- `email` 또는 `username`: 로그인 식별자.
- `passwordHash`: 해시된 비밀번호.
- `role`: 관리자 등 권한 구분.
- `createdAt`: 계정 생성 일시.
- `lastLoginAt`: 마지막 로그인 일시.

## UX 요구사항

- 첫 화면은 마케팅 랜딩 페이지가 아니라 실제 기기 관리 화면 또는 로그인 화면이어야 한다.
- 개인 도구답게 조용하고 실용적인 화면 구성을 우선한다.
- 데스크톱에서는 목록, 필터, 상세 정보를 효율적으로 탐색할 수 있어야 한다.
- 모바일에서는 한 손 조작, 카메라 등록, 복사 버튼 접근성을 우선한다.
- 주요 작업 버튼에는 목적이 분명한 아이콘과 텍스트를 사용한다.
- 입력 오류, 저장 성공, 삭제 완료, 카메라 권한 실패 같은 상태를 명확히 표시한다.
- 긴 QR 인식값이나 메모가 화면을 깨지 않도록 줄바꿈과 overflow 처리를 고려한다.

## 범위 밖

- 공개 회원가입과 다중 사용자 초대 기능.
- 클라우드 동기화.
- Matter 기기 제어 또는 페어링 자동화.
- 실제 API, DB 스키마, ORM, 서버 액션, 컴포넌트 구현.
- 테스트 프레임워크 확정.

## 수용 기준

- 로그인하지 않은 사용자는 저장된 Matter 기기 정보를 볼 수 없다.
- 로그인한 사용자는 Matter 기기를 등록, 조회, 수정, 삭제할 수 있다.
- QR 인식값과 숫자 코드는 수동 입력으로 등록할 수 있다.
- 지원 브라우저에서는 카메라 QR 스캔으로 QR 인식값을 등록할 수 있다.
- 저장된 QR 인식값은 화면에서 QR 코드로 표시된다.
- QR 인식값과 숫자 코드는 복사할 수 있다.
- SQLite DB 경로는 `MATTER_SQLITE_PATH`로 설정 가능하다.
- 앱 비밀값은 `AUTH_SECRET`로 설정 가능하다.
- 데스크톱과 모바일 화면에서 주요 작업이 겹치거나 잘리지 않는다.
- 요구사항 상세는 이 문서에 있고, `AGENTS.md`에는 이 문서 경로만 제공된다.
