# Matter Code Collector Mock 화면 설계

## 문서 목적

이 문서는 `docs/matter-code-collector/requirements.md`의 요구사항을 실제 mocking 화면으로 옮기기 위한 화면 설계 문서다. 아직 DB, API, 인증, QR 스캐너를 연결하지 않고 정적 샘플 데이터로 `src/app/page.tsx`에 구현할 수 있는 수준을 목표로 한다.

화면은 개인 운영 도구답게 조용하고 실용적인 관리 화면으로 설계한다. 첫 화면은 마케팅 랜딩이 아니라 로그인 상태에 따라 로그인 화면 또는 기기 목록 화면이 된다.

## 공통 디자인 방향

- Tailwind CSS v4 기반의 반응형 화면을 전제로 한다.
- 전체 톤은 흰색 또는 아주 옅은 회색 배경, 짙은 텍스트, 절제된 초록/청록 계열 포인트 컬러를 사용한다.
- 카드 반경은 작게 유지하고, 데이터 관리 화면답게 정보 밀도와 스캔 가능성을 우선한다.
- 데스크톱은 상단 앱 바, 좌측 필터 영역, 중앙 목록, 우측 상세 미리보기 구조를 기본으로 한다.
- 모바일은 상단 검색, 필터 칩, 카드형 목록, 하단 또는 우하단 주요 액션 버튼 구조를 기본으로 한다.
- 긴 QR 인식값과 메모는 줄바꿈, 축약, 복사 버튼을 함께 제공해 레이아웃을 깨지 않게 한다.
- 모든 primary action은 명확한 텍스트를 포함하고, 보조 action은 아이콘과 짧은 라벨을 함께 사용한다.

## 1. 로그인 화면

### 화면 목적

인증되지 않은 사용자가 저장된 Matter 기기 정보에 접근하기 전에 계정으로 로그인한다.

### 데스크톱 레이아웃

- 화면 중앙에 폭이 좁은 로그인 패널을 배치한다.
- 상단에는 앱 이름 `Matter Code Collector`와 짧은 설명 `개인 Matter QR 코드 보관함`을 표시한다.
- 패널에는 로그인 식별자, 비밀번호 입력, 로그인 버튼, 오류 메시지 영역이 있다.
- 하단에는 `공개 회원가입 없음`, `개인 운영 전용` 같은 작은 보조 문구를 둔다.

### 모바일 레이아웃

- 상단 여백을 줄이고 전체 폭에 가까운 로그인 패널을 사용한다.
- 입력과 버튼은 엄지로 누르기 쉬운 높이로 배치한다.
- 오류 메시지는 버튼 바로 아래에 한 줄 또는 두 줄로 표시한다.

### 주요 UI 요소

- 앱 타이틀
- 로그인 식별자 입력
- 비밀번호 입력
- 로그인 버튼
- 로그인 실패 상태 메시지
- 개인 운영 전용 안내 문구

### 사용자 흐름

1. 사용자가 로그인 식별자와 비밀번호를 입력한다.
2. 로그인 버튼을 누른다.
3. 성공 mock 상태에서는 기기 목록 화면으로 전환된 것처럼 보인다.
4. 실패 mock 상태에서는 `로그인 정보를 확인해 주세요.` 메시지를 표시한다.

### Mock 상태

- 기본 상태: 입력 필드가 비어 있고 버튼은 활성화되어 있다.
- 오류 상태: 비밀번호 오류 메시지를 표시한다.
- 로딩 상태: 로그인 버튼 텍스트를 `확인 중...`으로 바꾼다.

## 2. 기기 목록, 검색, 필터 화면

### 화면 목적

로그인한 사용자가 등록된 Matter 기기를 빠르게 찾고, QR 또는 숫자 코드를 확인할 기기를 선택한다.

### 데스크톱 레이아웃

- 상단 앱 바: 앱 이름, 검색 입력, `새 기기 등록`, `내보내기`, 사용자 메뉴.
- 좌측 필터 패널: 위치, 태그, 제조사, 정렬 옵션.
- 중앙 목록: 기기별 행 또는 compact card 목록.
- 우측 상세 미리보기: 선택된 기기의 QR 코드 영역, 숫자 코드, 복사 버튼, 수정/삭제 버튼.
- 화면 폭이 충분하면 목록과 상세 미리보기를 동시에 보여준다.

### 모바일 레이아웃

- 상단에 앱 이름과 검색 입력을 세로로 배치한다.
- 필터는 가로 스크롤 칩으로 제공한다.
- 목록은 카드형 단일 열로 표시한다.
- `새 기기 등록`은 하단 고정 버튼 또는 우하단 floating button으로 제공한다.
- 상세는 카드 탭 시 별도 상세 화면처럼 전체 폭으로 전환되는 mock 구조를 사용한다.

### 주요 UI 요소

- 검색 입력
- 위치/태그 필터 칩
- 정렬 선택: 최근 수정순, 최근 등록순, 이름순
- 기기 카드 또는 행
- 선택 상태 표시
- QR 미리보기 영역
- QR 값 복사 버튼
- 숫자 코드 복사 버튼
- 등록 데이터 내보내기 버튼

### 사용자 흐름

1. 사용자는 목록에서 기기를 훑어본다.
2. 검색어 또는 필터를 적용한다.
3. 기기를 선택하면 데스크톱은 우측 미리보기, 모바일은 상세 화면으로 이동하는 형태를 보여준다.
4. QR 값 또는 숫자 코드를 복사한다.
5. 필요하면 등록, 수정, 삭제, 내보내기 action으로 이동한다.

### Mock 상태

- 기본 상태: `mockDevices` 3개가 표시된다.
- 빈 상태: `등록된 Matter 기기가 없습니다.`와 `새 기기 등록` 버튼을 표시한다.
- 검색 결과 없음: `검색 조건과 일치하는 기기가 없습니다.`를 표시하고 필터 초기화 버튼을 둔다.
- 중복 경고 상태: 목록 상단 또는 등록 화면 연결 배너로 `동일한 QR 값이 이미 등록되어 있습니다.`를 표시한다.

## 3. 기기 등록 화면

### 화면 목적

사용자가 Matter 기기의 QR 인식값, 숫자 코드, 별명, 실제 기기 이름, 기타 정보를 등록한다.

### 데스크톱 레이아웃

- 화면은 2열 구성으로 잡는다.
- 좌측에는 입력 폼을 배치한다.
- 우측에는 카메라 스캔 패널, QR 값 미리보기, 입력 도움말을 배치한다.
- 저장 버튼과 취소 버튼은 폼 하단에 고정하거나 명확히 보이게 둔다.

### 모바일 레이아웃

- 단일 열 폼으로 구성한다.
- 상단에 `카메라로 스캔` 버튼을 크게 배치한다.
- QR 인식값, 숫자 코드, 별명은 우선 입력 필드로 노출한다.
- 상세 메타데이터는 아래쪽에 접힌 섹션처럼 보이게 구성할 수 있다.

### 주요 UI 요소

- `카메라로 스캔` 버튼
- QR 인식값 입력
- 숫자 코드 입력
- 기기 별명 입력
- 실제 기기 이름 입력
- 제조사, 모델, 위치 입력
- 태그 입력
- 메모 입력
- 저장 버튼
- 취소 버튼
- 중복 감지 경고

### 사용자 흐름

1. 사용자가 카메라 스캔 또는 수동 입력을 선택한다.
2. QR 인식값과 숫자 코드를 입력한다.
3. 별명과 실제 기기 이름을 입력한다.
4. 위치, 태그, 메모를 선택적으로 입력한다.
5. 저장을 누르면 목록에 새 기기가 추가된 것처럼 mock 상태를 보여준다.

### Mock 상태

- 기본 상태: 빈 입력 폼.
- 스캔 완료 상태: QR 인식값이 자동 채워지고 성공 배지를 표시한다.
- 중복 경고 상태: QR 인식값 아래에 중복 가능성 메시지를 표시한다.
- 저장 오류 상태: 필수값 누락 메시지를 필드 아래에 표시한다.

## 4. 카메라 QR 스캔 영역

### 화면 목적

브라우저 카메라로 Matter QR 코드를 읽어 등록 폼의 QR 인식값을 채운다.

### 데스크톱 레이아웃

- 등록 화면 우측 패널 안에 카메라 프리뷰 박스를 둔다.
- 프리뷰 박스 안에는 스캔 가이드 프레임과 상태 텍스트를 표시한다.
- 하단에는 `스캔 다시 시도`, `수동 입력으로 전환` 버튼을 제공한다.

### 모바일 레이아웃

- 전체 화면에 가까운 스캔 화면으로 표현한다.
- 상단에는 닫기 버튼과 제목을 둔다.
- 중앙에는 정사각형 스캔 프레임을 둔다.
- 하단에는 권한 상태와 수동 입력 버튼을 고정한다.

### 주요 UI 요소

- 카메라 프리뷰 영역
- 스캔 가이드 프레임
- 권한 요청 상태
- secure context 안내
- 수동 입력 fallback 버튼
- 스캔 성공 결과 표시

### 사용자 흐름

1. 사용자가 `카메라로 스캔`을 누른다.
2. 카메라 권한 상태를 확인한다.
3. 스캔 성공 시 QR 인식값을 등록 폼에 채운다.
4. 실패 또는 미지원 시 수동 입력으로 돌아간다.

### Mock 상태

- 권한 대기: `카메라 권한을 허용해 주세요.`
- 권한 거부: `카메라 접근이 거부되었습니다. 수동 입력을 사용해 주세요.`
- 미지원: `이 브라우저에서는 카메라 스캔을 사용할 수 없습니다.`
- insecure context: `카메라 스캔은 HTTPS 또는 localhost에서 사용할 수 있습니다.`
- 스캔 성공: 읽은 QR 인식값과 `등록 폼에 적용됨` 배지를 표시한다.

## 5. 기기 상세 화면

### 화면 목적

선택한 기기의 전체 정보를 확인하고 QR 코드, QR 인식값, 숫자 코드를 바로 사용할 수 있게 한다.

### 데스크톱 레이아웃

- 우측 상세 패널 또는 상세 페이지 상단에 큰 QR 코드 표시 영역을 둔다.
- QR 코드 옆에는 별명, 실제 기기 이름, 위치, 태그를 표시한다.
- 아래에는 QR 인식값, 숫자 코드, 제조사/모델, 메모, 등록/수정 일시를 섹션별로 배치한다.
- 상단 또는 우측에는 수정, 삭제, 복사 버튼을 제공한다.

### 모바일 레이아웃

- 상단에 별명과 위치를 표시한다.
- QR 코드 영역을 먼저 보여주고, 바로 아래에 숫자 코드와 복사 버튼을 둔다.
- 상세 메타데이터는 아래쪽 리스트로 배치한다.
- 수정과 삭제는 하단 action 영역에 둔다.

### 주요 UI 요소

- QR 코드 표시 영역
- QR 인식값 복사 버튼
- 숫자 코드 복사 버튼
- 별명, 실제 기기 이름
- 위치, 태그
- 제조사, 모델
- 메모
- 등록 일시, 수정 일시
- 수정 버튼
- 삭제 버튼

### 사용자 흐름

1. 사용자가 목록에서 기기를 선택한다.
2. QR 코드와 숫자 코드를 확인한다.
3. 필요한 값을 복사한다.
4. 정보가 틀리면 수정으로 이동한다.
5. 더 이상 필요 없으면 삭제 확인을 연다.

### Mock 상태

- 기본 상태: 선택된 첫 번째 mock 기기의 상세 정보 표시.
- 복사 완료 상태: 버튼 옆에 `복사됨` 피드백 표시.
- QR 렌더 실패 상태: QR 영역에 `QR 코드를 표시할 수 없습니다.`와 원본 문자열 복사 버튼 표시.

## 6. 기기 수정 화면

### 화면 목적

등록된 기기의 정보를 수정한다. QR 인식값과 숫자 코드는 실수 방지를 위해 변경 사실을 명확히 보여준다.

### 데스크톱 레이아웃

- 등록 화면과 같은 2열 구조를 재사용한다.
- 좌측에는 기존 값이 채워진 폼을 둔다.
- 우측에는 현재 QR 코드 미리보기와 변경 전/후 비교 영역을 둔다.

### 모바일 레이아웃

- 단일 열 폼으로 구성한다.
- QR 인식값과 숫자 코드 변경 필드는 상단에 경고 배지를 표시한다.
- 저장/취소 버튼은 하단에 명확히 둔다.

### 주요 UI 요소

- 기존 값이 채워진 입력 폼
- QR 인식값 변경 경고
- 숫자 코드 변경 경고
- 저장 버튼
- 취소 버튼
- 변경 사항 요약

### 사용자 흐름

1. 사용자가 상세 화면에서 수정 버튼을 누른다.
2. 기존 정보가 채워진 폼을 확인한다.
3. 필요한 필드를 수정한다.
4. QR 인식값 또는 숫자 코드가 바뀌면 변경 경고를 확인한다.
5. 저장하면 상세 화면으로 돌아간 것처럼 표시한다.

### Mock 상태

- 기본 상태: 두 번째 mock 기기의 값이 채워져 있다.
- 변경 감지 상태: 변경된 필드에 강조 표시.
- 저장 오류 상태: 필수값 누락 또는 중복 경고 표시.

## 7. 삭제 확인 모달

### 화면 목적

기기 삭제 전 사용자가 대상 기기를 명확히 확인하고 실수 삭제를 줄인다.

### 데스크톱 레이아웃

- 화면 중앙에 작은 모달을 띄우고 뒤 배경은 약하게 어둡게 처리한다.
- 모달에는 별명, 실제 기기 이름, 숫자 코드 일부를 표시한다.
- `삭제` 버튼은 위험 action 색상, `취소` 버튼은 보조 스타일로 둔다.

### 모바일 레이아웃

- 하단 sheet 형태로 표시한다.
- 대상 기기 정보를 먼저 보여주고, 버튼은 세로로 충분한 높이를 확보한다.

### 주요 UI 요소

- 삭제 대상 요약
- 삭제 경고 문구
- 취소 버튼
- 삭제 버튼

### 사용자 흐름

1. 사용자가 상세 화면에서 삭제를 누른다.
2. 삭제 확인 모달이 열린다.
3. 취소하면 이전 화면으로 돌아간다.
4. 삭제하면 목록에서 제거된 것처럼 mock 상태를 보여준다.

### Mock 상태

- 기본 상태: 삭제 대상 정보 표시.
- 삭제 진행 상태: `삭제 중...`
- 삭제 완료 상태: 목록 빈 상태 또는 목록 갱신 상태 표시.

## 8. 백업 및 내보내기 진입 UI

### 화면 목적

등록 데이터를 개인 백업 용도로 내보낼 수 있는 진입점을 제공한다.

### 데스크톱 레이아웃

- 상단 앱 바 또는 목록 화면 우측 상단에 `내보내기` 버튼을 둔다.
- 클릭 시 작은 패널 또는 모달에서 내보내기 형식을 선택하는 mock UI를 보여준다.

### 모바일 레이아웃

- 사용자 메뉴 또는 목록 상단 보조 action에 `내보내기`를 둔다.
- 선택 패널은 하단 sheet 형태로 표시한다.

### 주요 UI 요소

- 내보내기 버튼
- JSON 내보내기 옵션
- CSV 내보내기 옵션
- 마지막 백업 일시 표시
- SQLite DB 경로가 환경변수로 관리된다는 운영 안내 문구

### 사용자 흐름

1. 사용자가 목록 화면에서 내보내기를 누른다.
2. 형식을 선택한다.
3. mock 상태에서는 `내보내기 파일 준비됨` 메시지를 보여준다.

### Mock 상태

- 기본 상태: JSON, CSV 옵션 표시.
- 준비 완료: `백업 파일이 준비되었습니다.`
- 오류 상태: `내보내기를 준비하지 못했습니다. 다시 시도해 주세요.`

## Mocking 구현 기준

- 실제 DB, API, 인증, QR 스캐너, 파일 다운로드는 연결하지 않는다.
- 모든 화면은 정적 데이터와 로컬 UI 상태만으로 표현한다.
- `src/app/page.tsx`에서 한 화면에 dashboard mock을 구현해도 되고, 내부 상태로 로그인/목록/등록/상세/수정/삭제 모달을 전환해도 된다.
- QR 코드는 실제 생성 라이브러리를 붙이기 전에는 사각형 패턴 placeholder로 표시해도 된다.
- 카메라 프리뷰는 실제 비디오 대신 어두운 preview placeholder와 스캔 프레임으로 표현한다.
- 아이콘이 필요하면 구현 단계에서 `lucide-react` 같은 아이콘 라이브러리 사용 여부를 결정한다.
- 화면 텍스트는 한국어를 기본으로 한다.

## 구현자가 확인할 수용 기준

- 로그인, 목록, 등록, 카메라 스캔, 상세, 수정, 삭제 확인, 내보내기 UI가 모두 mock으로 표현된다.
- 데스크톱 폭에서는 필터, 목록, 상세 미리보기가 한 화면에서 스캔 가능하다.
- 모바일 폭에서는 검색, 필터 칩, 카드 목록, 주요 action이 겹치지 않는다.
- QR 인식값과 숫자 코드는 긴 문자열이어도 레이아웃을 깨지 않는다.
- 카메라 권한 거부, secure context 안내, 수동 입력 fallback 상태가 화면에 존재한다.
- SQLite와 인증은 실제 구현되지 않지만, `MATTER_SQLITE_PATH`, `AUTH_SECRET`, 로그인 전제는 운영 안내 또는 상태 문구에 반영된다.
